⚙️ Hacking do EmuELEC: Guia SquashFS

Processo de 5 passos para desmontar, modificar e recompactar o arquivo **SYSTEM** (que é o SquashFS principal) do EmuELEC.

⚠️ ATENÇÃO: Pré-requisitos e Risco

  • Você deve executar este processo em um ambiente **Linux**.
  • Você deve ter os pacotes essenciais instalados. Se o comando fakeroot falhar, execute: sudo apt install squashfs-tools fakeroot
  • O uso de fakeroot é **CRÍTICO** para preservar permissões de root.

1. Diagnóstico: Confirmação dos Parâmetros (JÁ COMPLETO)

Você já obteve a informação crítica. Seus parâmetros são: 

$ unsquashfs -s SYSTEM | grep -E 'Compression|Block size'

Seus Parâmetros:
Compression: lzo
Block size: 524288

2. Extração com Preservação de Root (JÁ CONCLUÍDO)

O comando fakeroot unsquashfs foi concluído com sucesso. Agora o conteúdo está no diretório squashfs-root. 

$ fakeroot unsquashfs -f -d squashfs-root SYSTEM

3. Modificação do Overlay Customizado (Sua Vez)

Agora você pode entrar no diretório squashfs-root para adicionar ou editar seus arquivos. 

# Entra no diretório
$ cd squashfs-root

# Realize suas modificações aqui. Exemplo:
# $ mkdir -p storage/downloads
# $ cp /caminho/do/meu/script.sh usr/bin/

# Retorna ao diretório principal antes de recompactar
$ cd ..

Ação: Faça suas alterações agora. Quando terminar, volte para o diretório `COMPILAR_EMUELEC` e prossiga para o Passo 4.

4. Recompactação (Com Seus Parâmetros Exatos)

Comando configurado com **`lzo`** e **`524288`**. Este comando é o mais importante para evitar falhas de inicialização. 

$ fakeroot mksquashfs squashfs-root SYSTEM \
    -comp lzo \
    -b 524288 \
    -no-xattrs -all-root -noappend

Observação: Este processo pode levar alguns minutos, dependendo da velocidade do seu sistema e do tamanho do arquivo `SYSTEM`.

5. Implantação e Teste

Após a recompactação, você terá o arquivo **SYSTEM_NEW**.

  1. Renomeie: Renomeie o arquivo de SYSTEM_NEW para SYSTEM.
  2. Backup: Faça backup do arquivo SYSTEM original no seu dispositivo.
  3. Substitua: Copie o novo arquivo para a partição **BOOT (EMUELEC)** do seu dispositivo.
  4. Teste: Reinicie o EmuELEC.
Guia baseado em Engenharia Reversa de Sistemas EmuELEC/LibreELEC.
